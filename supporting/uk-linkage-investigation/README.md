# uk-linkage-investigation

An **evidence-first, non-accusatory** documentation project on UK public-sector
**probabilistic record linkage** — principally the Ministry of Justice's open-source
[Splink](https://github.com/moj-analytical-services/splink) library and the linkage
programmes that use it (MoJ *Data First*, NHS England's *Data Linkage Hub*, ONS
Census→PDS linkage).

## Purpose

Document, from primary sources, four neutral questions:

1. **What** personal data is linked?
2. **By whom**, and shared with whom?
3. Under **what legal basis** (UK GDPR Art. 6 / Art. 9 / Art. 10; Data Protection Act 2018)?
4. With **what safeguards**, and **what rights** do citizens have (access, objection, opt-out)?

We assemble sourced facts, mark our confidence honestly, and turn the gaps into
Freedom of Information requests. **We do not allege wrongdoing.** Where the public
record is silent, the honest answer is `UNKNOWN`, not an inference.

## Status taxonomy (exactly one per claim)

| Status | Meaning | Evidence bar |
|--------|---------|--------------|
| **CONFIRMED** | Established fact | Primary / official source cited and live |
| **LIKELY** | Strong inference | Good secondary source, or sound inference from primary sources |
| **UNKNOWN** | No sourced answer | Becomes a TODO / FOI request — never guessed |
| **SPECULATION** | Explicit hypothesis | Clearly labelled; never presented as fact |

## Workflow

1. Every source goes into `evidence/raw_links.json` **and** `research/sources.md`.
2. `python scripts/source_checker.py` fetches each URL and classifies it
   (DEAD / REDIRECT / STALE / UNCLEAR / OK). It exits non-zero on any DEAD link.
3. Only **live, verified** sources may back a CONFIRMED or LIKELY claim in
   `research/claims_matrix.md`. Unsourced assertions are UNKNOWN and flow to
   `research/open_questions.md` → `foi/foi_requests.md`.
4. Statutory references are checked against the current UK GDPR / Data Protection
   Act 2018 text on legislation.gov.uk before being recorded.

## Branch discipline

Research is done on `research/first-pass`. `main` is kept clean until Phase 5.
No force-push, no history rewrite.
